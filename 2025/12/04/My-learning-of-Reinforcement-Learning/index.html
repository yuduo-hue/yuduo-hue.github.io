My Reinforcement Learning Journey

Created2025-12-04|Updated2025-12-04
|Post Views:

“Learning RL felt like assembling a machine while it was already running.”
Over the past weeks I moved from casual curiosity about reinforcement learning (RL) to building small projects in Gymnasium, and then to probing the mathematical foundations that justify the code. This post records what I’ve learned—concepts, experiments, and insights—so future-me (and other beginners) can avoid some of my confusion.

1) Orientation: what I thought RL was—and what it really is

At first, RL seemed like supervised learning with a twist. In reality, RL formalizes sequential decision-making under uncertainty. We model the world as a Markov Decision Process (MDP) with a state space, an action set, a transition rule, and a reward signal. The guiding star is the return, a discounted sum of future rewards:

Gt=∑k=0∞γkRt+1+k,0≤γ<1.G_t = \sum_{k=0}^{\infty}\gamma^k R_{t+1+k}, \quad 0\le \gamma < 1. Gt​=k=0∑∞​γkRt+1+k​,0≤γ<1.

Two functions organize our thinking:

  • State-value (V^\pi(s)): expected return from state (s) following policy (\pi).
  • Action-value (the famous “Q”):

    Qπ(s,a)=Eπ ⁣[∑k=0∞γkRt+1+k | St=s,At=a].Q^\pi(s,a)=\mathbb{E}_\pi\!\left[\sum_{k=0}^{\infty}\gamma^k R_{t+1+k}\,\middle|\,S_t=s, A_t=a\right]. Qπ(s,a)=Eπ​[k=0∑∞​γkRt+1+k​​St​=s,At​=a].

Plainly: Q(s,a) estimates “how good it is to take action (a) in state (s), considering all that follows.”

My first working agent learned on Blackjack. The environment’s state is compact—((\text{player_sum},\ \text{dealer_card},\ \text{usable_ace}))—so a simple Q-table works well.

I implemented an (\epsilon)-greedy policy: explore with probability (\epsilon); otherwise exploit the current best action. The Q-learning update in my code mirrors the Bellman optimality idea:

In words: move your estimate toward immediate reward + discounted best future value.

What “off-policy” clicked for me

Q-learning is off-policy: I behave (\epsilon)-greedily (sometimes random) but learn as if I always took the greedy action next. This gap—behavior policy vs target policy—explains why Q-learning often converges to a sharper, more “idealized” solution than what it actually did during exploration.

3) A contrast that sharpened my intuition: SARSA vs Q-learning

  • SARSA (on-policy) updates with the action I actually chose next:

    Q(St,At)←Q(St,At)+α(Rt+1+γQ(St+1,At+1)−Q(St,At)).Q(S_t,A_t)\leftarrow Q(S_t,A_t)+\alpha\bigl(R_{t+1}+\gamma Q(S_{t+1},A_{t+1})-Q(S_t,A_t)\bigr). Q(St​,At​)←Q(St​,At​)+α(Rt+1​+γQ(St+1​,At+1​)−Q(St​,At​)).

    It learns the value of the current exploratory policy—often safer in risky environments.

  • Q-learning (off-policy) updates with the best possible next action:

    Q(St,At)←Q(St,At)+α(Rt+1+γmax⁡aQ(St+1,a)−Q(St,At)).Q(S_t,A_t)\leftarrow Q(S_t,A_t)+\alpha\bigl(R_{t+1}+\gamma\max_{a}Q(S_{t+1},a)-Q(S_t,A_t)\bigr). Q(St​,At​)←Q(St​,At​)+α(Rt+1​+γamax​Q(St+1​,a)−Q(St​,At​)).

    It learns the value of the greedy target policy, even while behaving non-greedily.

This distinction finally made sense when I imagined “cliff walking”: SARSA learns a cautious path (aware of exploration), while Q-learning imagines the ideal greedy path (which can be riskier during learning).

5) Where the math justifies the code

The Bellman equations explain why “backup” targets are recursive. For the optimal action-value function,

Q∗(s,a)=E[Rt+1+γmax⁡a′Q∗(St+1,a′)∣St=s,At=a].Q^*(s,a)=\mathbb{E}\bigl[R_{t+1}+\gamma\max_{a'}Q^*(S_{t+1},a') \mid S_t=s,A_t=a\bigr]. Q∗(s,a)=E[Rt+1​+γa′max​Q∗(St+1​,a′)∣St​=s,At​=a].

My update is a stochastic approximation to this fixed point. Repeatedly nudging the table toward these targets (with a prudent learning rate) is what eventually stabilizes behavior.

Understanding expectations, conditioning, and discounting turned the code from a recipe into a rationale. The algorithm is no longer a black box; it is a disciplined approximation to a mathematically defined object.

6) What I’m still refining—and how I’ll study next

  • Discretization bias: binning is convenient but crude. Next I’ll explore function approximation (e.g., DQN) to handle continuous states more gracefully.
  • Exploration schedules: I want a more principled decay for (\epsilon) and perhaps try softmax policies.
  • Evaluation: beyond average return, I’ll examine variance, stability, and sensitivity to hyperparameters.

My study routine (concise)

  1. Skim a small theory section (value functions, Bellman, TD).
  2. Watch the corresponding lecture clip for intuition.
  3. Translate one formula directly into code comments.
  4. Run a tiny experiment; record an observation in plain English.

7) Personal reflection (and writing practice)

Writing these notes in English forced me to be precise yet readable. I intentionally adopted more mature vocabulary—intuitive, principled, approximation, stabilize, rationale—to cultivate academic clarity. This aligns with my CET-4 preparation: not merely avoiding mistakes, but choosing stronger words and varying sentence rhythm.

Appendix: Mini glossary (my own words)

  • Return: discounted sum of future rewards; a long-term score.
  • Policy: a rule mapping states to action probabilities.
  • On-policy vs Off-policy: learn about the policy you execute vs a different (usually greedier) policy.
  • Temporal-Difference (TD): learn from bootstrapped targets using current estimates.
  • Bellman optimality: the equation that defines the best achievable value by looking one step ahead plus the optimal future.
