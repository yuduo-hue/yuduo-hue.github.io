SimpleGridEnv: A Minimal Custom Gymnasium Environment

This little SimpleGridEnv is a minimal custom Gymnasium environment: a 5×5 grid where an agent moves around, gets -1 per step, and +10 when it reaches the goal.

I’ll explain it in plain language first, then list official docs you can read.

1. What This Environment Is

  • World: a 5×5 grid
  • State: a single integer from 0 to 24 (flattened cell index)
  • Start: bottom-left cell
  • Goal: top-right cell
  • Actions:
    • 0: up
    • 1: right
    • 2: down
    • 3: left
  • Reward:
    • +10 when you reach the goal
    • -1 every other step
  • Episode ends when:
    • the agent reaches the goal (terminated = True)

So you’ve created a tiny RL “maze” to test algorithms.

2. Line-by-Line Explanation

  • gymnasium – the RL environment API
  • spaces – defines valid observation and action formats (Discrete, Box, etc.)
  • numpy – used here only to build a small grid for printing

Class Definition and Metadata

  • You subclass gym.Env, which is the base class all Gymnasium environments must follow
  • metadata tells Gym what render modes the env supports. Here you only support "human" (printing to the console)

__init__: Define Spaces and Basic Properties

  • self.size = 5 → grid is 5×5
  • self.observation_space = spaces.Discrete(25):
    • Agent’s observation is a single integer 0..24
    • Each number corresponds to a grid cell (flattened index)
  • self.action_space = spaces.Discrete(4):
    • There are 4 possible actions, numbered 0–3
  • self.state will store the current cell index

This matches the Gymnasium requirement that every env has observation_space and action_space.

reset: Start a New Episode

  • reset() is called at the start of each episode
  • super().reset(seed=seed) lets Gymnasium handle seeding
  • self.state = (self.size - 1) * self.size + 0:
    • self.size - 1 is the last row (row index 4 in a 0–4 grid)
    • Multiply by self.size to flatten (row, col) → index:
      • bottom-left = row 4, col 0 → 4*5 + 0 = 20
  • You return:
    • observation = self.state
    • info = {} (an empty dict, for extra debug info if needed)

step: Apply an Action and Move the Agent

  • divmod(self.state, self.size) converts a flat index back to 2D:
    • row = self.state // 5
    • col = self.state % 5

Then update the position based on the action, but don’t walk off the grid:

  • action == 0 → go up (row - 1), but only if not already at top edge (row > 0)
  • action == 1 → go right (col + 1), but only if col < 4
  • action == 2 → go down (row + 1), but only if row < 4
  • action == 3 → go left (col - 1), but only if col > 0
  • If the action would go outside, you simply ignore it and stay in place

Update the flattened state again:

Now define the goal and termination condition:

  • Goal is top-right:
    • row 0, col 4 → 0 * 5 + 4 = 4
  • terminated:
    • True if you reached the goal cell
    • RL episode ends because the task is solved
  • truncated:
    • False here (you’re not using a time limit)
    • In more complex environments, truncated=True means “ended due to time limit or external reason, not a natural terminal state”

Finally, compute reward:

  • If you reached the goal → big positive reward +10
  • Otherwise → small penalty -1 to encourage the shortest path

And return the full result (Gymnasium’s step contract):

  • Next observation = self.state
  • Scalar reward
  • terminated flag
  • truncated flag
  • info dict (empty for now)

render: Show the Grid in the Terminal

  • Creates a 5×5 array filled with "."
  • Places "A" at the agent’s position (r, c)
  • Prints the grid row by row
  • This is a super simple “human render”; in real projects you might use matplotlib or a game engine instead

    Explains exactly how to subclass gymnasium.Env, define observation_space, action_space, and implement reset / step / render.

  2. Gymnasium Env API

  3. Gym / Gymnasium environment creation (older but still useful)
    Very similar concepts, includes a gridworld example and explains environment lifecycle.

  4. Spaces API
    Documentation for spaces.Discrete, spaces.Box, and other space types used to describe observations and actions.

These will give you both official references and concrete examples similar to your SimpleGridEnv.
